Etiquetas para formularios

Las etiquetas más comunes para crear formularios HTML que permiten la recopilación de datos del usuario.

<form> </form>

Define un formulario HTML para la recopilación de datos del usuario. Acepta los siguientes atributos:

  • action: URL o path a donde se envía la información del formulario al presionar el botón de enviar.
  • method: método HTTP que se utiliza al enviar el formulario (GET o POST).
  • enctype: especifica cómo se codifican los datos del formulario al enviarlos al servidor. Valores comunes: application/x-www-form-urlencoded (por defecto), multipart/form-data (necesario para enviar archivos) y text/plain.
  • target: especifica dónde se muestra la respuesta del servidor después de enviar el formulario. Valores: _self (misma pestaña, por defecto), _blank (nueva pestaña), _parent y _top.
HTML 
<form action="/submit" method="POST" enctype="multipart/form-data" target="_blank">
  ...contenido del formulario...
</form>

<label> </label>

Define una etiqueta para un elemento de formulario. Usa el atributo for para asociar la etiqueta con un elemento de formulario específico, por ejemplo:

HTML 
<label for="nombre">Nombre:</label>

<input>

Define un campo de entrada para la recopilación de datos del usuario. Los distintos tipos se pueden consultar en MDN Web Docs. Acepta los siguientes atributos:

  • type: tipo de campo de entrada (texto, contraseña, correo electrónico, etc.).
  • id: su valor debe ser igual al suministrado en el atributo for del label.
  • name: identifica el campo de entrada al enviar el formulario.
  • required: indica que el campo es obligatorio.
  • placeholder: texto de sugerencia que se muestra en el campo.
  • value: valor predeterminado para el campo.
  • autocomplete: habilita o deshabilita el autocompletado del navegador.
  • min y max: límites en los valores numéricos.
  • step: incremento permitido para los valores numéricos.
  • pattern: expresión regular que el valor del campo debe cumplir.
  • autofocus: el campo recibe el foco automáticamente al cargar la página.
HTML 
<input type="text" name="nombre" placeholder="Nombre" required />

Tipos más comunes

Tipo Descripción
text Texto de una sola línea
email Correo electrónico (valida formato)
password Contraseña (oculta caracteres)
number Valor numérico con controles de incremento
date Selector de fecha
checkbox Casilla de verificación (selección múltiple)
radio Botón de opción (selección única)
file Selector de archivos
submit Botón para enviar el formulario

Ejemplo completo de formulario

Ejemplo de un formulario con <form>, <label> e <input>:

HTML 
<form action="/submit" method="POST">
  <label for="nombre">Nombre:</label>
  <input type="text" id="nombre" name="nombre" placeholder="Nombre" required />

  <label for="email">Email:</label>
  <input type="email" id="email" name="email" placeholder="Email" required />

  <input type="submit" value="Enviar" />
</form>
Resultado

<fieldset> </fieldset>

Funciona como contenedor de los inputs, colocando un borde alrededor del grupo. Se utiliza junto con la etiqueta <legend> para definir un título para el grupo de campos, por ejemplo:

HTML 
<fieldset>
  <legend>Elige los cursos a los que deseas asistir:</legend>

  <input type="checkbox" id="html" name="html" />
  <label for="html">HTML básico</label>

  <input type="checkbox" id="css" name="css" />
  <label for="css">CSS avanzado</label>

  <input type="checkbox" id="js" name="js" />
  <label for="js">JavaScript</label>
</fieldset>
Resultado

<textarea> </textarea>

Define un área de texto para la entrada de texto multilínea. Acepta los siguientes atributos:

  • name: identifica el área de texto al enviar el formulario.
  • rows: número de filas visibles en el área de texto.
  • cols: número de columnas visibles en el área de texto.
  • placeholder: texto de sugerencia que se muestra en el área de texto.
HTML 
<textarea name="mensaje" rows="4" cols="50" placeholder="Escribe tu mensaje aquí..."></textarea>
Resultado

<select> </select>

Define un menú desplegable (dropdown) con múltiples opciones. Se utiliza junto con la etiqueta <option> para definir cada opción disponible, y <optgroup> para agrupar opciones relacionadas. Atributos importantes:

  • name: identifica el campo al enviar el formulario.
  • multiple: permite seleccionar múltiples opciones.
  • size: número de opciones visibles a la vez.
  • required: hace obligatoria la selección de una opción.
HTML 
<label for="pais">País:</label>
<select name="pais" id="pais" required>
  <option value="">-- Selecciona un país --</option>
  <optgroup label="América">
    <option value="mx">México</option>
    <option value="ar">Argentina</option>
  </optgroup>
  <optgroup label="Europa">
    <option value="es">España</option>
    <option value="fr">Francia</option>
  </optgroup>
</select>
Resultado

<button> </button>

Define un botón clicable. A diferencia de <input type="submit">, permite incluir contenido HTML como iconos, imágenes o spans. Acepta los siguientes atributos:

  • type: tipo de botón (submit, reset, button).
  • name: identifica el botón al enviar el formulario.
  • value: valor enviado cuando se presiona el botón.
  • disabled: deshabilita el botón.
HTML 
<button type="submit">Enviar formulario</button>
<button type="reset">Limpiar</button>
<button type="button">Acción personalizada</button>
Resultado

<datalist> </datalist>

Proporciona una lista de opciones predefinidas para un <input>, creando un autocompletado. El usuario puede elegir una opción de la lista o escribir un valor personalizado. Se vincula usando el atributo list del input con el id del datalist.

HTML 
<label for="navegador">Elige tu navegador:</label>
<input type="text" id="navegador" name="navegador" list="navegadores" />

<datalist id="navegadores">
  <option value="Chrome">
  <option value="Firefox">
  <option value="Safari">
  <option value="Edge">
  <option value="Opera">
</datalist>
Resultado
